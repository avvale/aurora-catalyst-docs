How-to guides are **task-oriented**: they assume you already understand the fundamentals and just need to solve a specific problem.

If you are new to Aurora, start with the [tutorials](/tutorials/) instead.

## Categories

- **CLI** — working with the `catalyst` command (generate, provision, deploy, env, sheets).
- **Framework** — customizing the generated Angular + NestJS code.
- **Preservation regions** — keeping your hand-written edits safe across regenerations.
