## Goal

Configure a Catalyst app so it delegates **all** login to an external Aurora Hub, using the OAuth 2.1 Authorization Code flow with PKCE. The app keeps no login form of its own â€” unauthenticated users are bounced to the hub, and the hub issues the tokens.

For the roles and the token flow behind these steps, see [Authentication strategies](/aurora-catalyst-docs/en/concepts/backend/authentication-strategies/).

## Before you start

- A **running Aurora Hub** â€” a Catalyst instance in `local-provider` mode â€” reachable from both the browser and the satellite backend. It must expose `GET /api/o-auth/authorize`, `POST /api/o-auth/token`, `POST /api/o-auth/session`, `GET /.well-known/jwks.json`, and `/graphql`.
- The hub's `OAuthApplication` **code** (e.g. `aurora`) and its **secret** (the value the hub seeds via `BOOTSTRAP_OAUTH_APP_SECRET`).
- A satellite generated by a recent CLI â€” it ships the `authorization-code` BFF package and the `authStrategy` front gating. If your app predates them, re-pull with `catalyst add back --force`.
- The URLs you will use. This guide assumes the hub at `http://localhost:8080`, the satellite frontend at `http://localhost:4201`, and the satellite backend at `http://localhost:8081`. Adjust to your deployment.

## Steps

### 1. Register an Authorization Code client on the hub

In the hub admin, create an **OAuthClient** (the *Edit client* dialog):

1. **Grant Type** â†’ `Authorization Code`.
2. **Applications** â†’ the application the satellite belongs to (e.g. `Aurora`). This links the client to the `OAuthApplication` whose `code` the satellite will send.
3. **Redirect** â†’ the satellite's callback, **exactly**: `http://localhost:4201/callback`. It must be absolute `http(s)`, with no trailing fragment (`#`). The hub matches this string character-for-character at both `/authorize` and the token exchange.
4. Set token lifetimes and **Active = on**, then save.

:::caution[Which secret the satellite uses]
The satellite authenticates with the **application's** `code` and `secret` â€” *not* the `OAuthClient` UUID and *not* the client's own "Secret" field in this dialog (that one is vestigial for the Authorization Code grant). `client_id` is the `OAuthApplication.code` (`aurora`); `client_secret` is the `OAuthApplication.secret`.
:::

:::note
A fresh hub's bootstrap seeder only creates the **Password** client. Add the `AUTHORIZATION_CODE` client yourself (admin UI or GraphQL) as above.
:::

### 2. Generate the RSA key pair on the satellite

```bash
catalyst keys --out ./backend/.keys/
```

This writes `oauth-private.key` and `oauth-public.key`. Hub mode validates tokens against the hub's JWKS and never signs anything â€” but the backend still reads the private key at boot and crashes without it, so this step is required even here. See [`catalyst keys`](/aurora-catalyst-docs/en/reference/cli-commands/keys/).

### 3. Configure the satellite backend `.env`

In `backend/.env`:

| Variable | Example | What it is |
| --- | --- | --- |
| `OAUTH_STRATEGY` | `aurora-hub` | Selects hub mode. The default is `none` â€” you must set this. |
| `OAUTH_HUB_SERVER_URL` | `http://localhost:8080` | Base URL of the hub backend. |
| `OAUTH_APPLICATION_CODE` | `aurora` | The hub `OAuthApplication.code`. Sent as `client_id`. |
| `OAUTH_APPLICATION_SECRET` | `'$2y$10$EOA/â€¦'` | The hub `OAuthApplication.secret`. Used only in the backend's HTTP Basic header â€” never shipped to the frontend. |
| `OAUTH_REDIRECT_URI` | `http://localhost:4201/callback` | The satellite callback. Must equal the hub client's **Redirect** exactly. |
| `APP_FRONTEND_URL` | `http://localhost:4201` | Base URL of the satellite frontend. |
| `OAUTH_PRIVATE_KEY_PATH` | `./.keys/oauth-private.key` | Defaults to this â€” only set it if you store keys elsewhere. |

:::caution[The `$` gotcha]
The application secret is a bcrypt-format string containing `$`. Wrap it in **single quotes** in `.env`; in `docker-compose` escape each `$` as `$$`. After loading, confirm the value is 60 characters â€” a truncated secret yields `401 invalid_client`.
:::

### 4. Point the satellite frontend at hub mode

In your active environment file (`frontend/src/environments/environment.ts`):

- Set **`authStrategy: 'authorization-code'`**. This single field is the source of truth: it selects the code guard, drops the local login form, and routes sign-out to a terminal page. The scaffold default is `'password'`, so you must flip it.
- Set **`api.rest`** to the satellite's own backend (the BFF), e.g. `http://localhost:8081/api` â€” **not** the hub. The `/callback` round-trip POSTs to this backend.

The `/callback` route ships pre-wired and unguarded; do not add an auth guard to it.

### 5. Restart and run

Restart the satellite backend (the strategy is read once at boot), then start both apps.

## Verify it worked

Open a protected route on the satellite and watch the round-trip:

1. The browser is redirected to the satellite backend `â€¦/api/auth/login`, then 302s to the hub `â€¦/api/o-auth/authorize` with `code_challenge` and `code_challenge_method=S256`.
2. You log in on the **hub**, which redirects to `http://localhost:4201/callback?code=â€¦&state=â€¦`.
3. The callback POSTs to the satellite `â€¦/api/auth/token`, tokens are stored, and you land authenticated.

Green checklist: no `invalid_request` (PKCE travelled), no call to `/credentials` (legacy gone), no `401` (the application secret matched), no redirect loop, and sign-out reaches the terminal page without bouncing back to the hub.

## Troubleshooting

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| `400 invalid_request` | PKCE missing, or `client_id` is a UUID (legacy BFF) | Re-pull the BFF (`catalyst add back --force`); set `OAUTH_APPLICATION_CODE` to the application `code`. |
| `400 redirect_uri mismatch` | Hub client **Redirect** â‰  `OAUTH_REDIRECT_URI` | Make them identical â€” absolute, no fragment. |
| `401 invalid_client` | `OAUTH_APPLICATION_SECRET` wrong or truncated | Use the application secret; mind the `$` quoting; confirm 60 chars. |
| `429 ThrottlerException` | A redirect loop hammering the hub | Fix the underlying 4xx, then restart the hub (the throttle counter is in-memory). |
| Blank page / `NG04002` | Hub sign-in path misconfigured | Check the hub's `OAUTH_SIGN_IN_PATH` (default `/auth/sign-in`). |
